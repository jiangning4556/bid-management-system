# 投标管理系统 - 技术方案设计

## 一、项目概述

### 1.1 项目背景
本系统是一个企业级的投标管理平台，用于管理咨询项目、供应商报价、中标项目、收支记录等业务流程。

### 1.2 核心功能模块
- 用户认证与权限管理
- 分类与物品管理
- 咨询项目管理
- 中标项目管理
- 供应商管理
- 收支管理
- 价格查询
- 统计报表
- 文件管理
- 日志管理
- 提醒管理

## 五、核心业务逻辑

### 5.1 转为中标项目流程

```
1. 用户在咨询项目详情页点击"转为中标项目"按钮
2. 前端验证是否有选中的供应商报价
3. 调用 POST /api/bid-projects/from-consult/:consultId
4. 后端执行:
   a. 查询咨询项目及其项目项、报价
   b. 创建新的中标项目
   c. 复制有选中报价的项目项
   d. 复制选中的供应商报价到中标供应商
   e. 计算项目总金额
5. 返回创建的中标项目信息
6. 前端跳转到中标项目详情页
```

**关键约束：**
- 只有选中的供应商报价会被复制到中标项目
- 没有选中报价的项目项会被跳过
- 中标项目保存 `consultProjectId` 关联到源咨询项目

### 5.2 供应商统计数据动态计算

供应商表中的统计数据是动态计算的：
- `projectCount`: 参与的咨询项目数量
- `bidProjectCount`: 中标的项目数量
- `bidRate`: 中标率 = bidProjectCount / projectCount * 100

**计算时机：** 每次查询供应商详情时计算

### 5.3 Excel导出功能

**权限控制：** 只有管理员角色可以导出Excel

**实现方式：**
- 后端使用 ExcelJS 生成Excel文件
- 返回 blob 类型的响应
- 前端使用 blob 方式下载文件
